## Phase 0: 需求分析与架构规划

=== 项目启动 ===
- 项目名称: 白酒供应链协同平台 - 供应商全链路协同
- 分5大阶段: 供应商准入 / 寻源合同 / 订单执行 / 财务结算
- 当前Phase: Phase 1 (供应商准入)
- 已有资产: React前端(Ant Design)、FastAPI后端、SQLite数据库

=== 架构决策 ===
- 采用前后端分离架构
  - 前端: React + Ant Design + Vite (现有)
  - 后端: FastAPI + SQLAlchemy + SQLite (现有)
  - 协同API: /api/supplier-portal/*, /api/qualification/*, /api/sourcing/*, /api/forecast/*, /api/asn/*, /api/settlement/*, /api/invoices/*, /api/payments/*
- 数据库: 复用现有SQLite，新增协同相关表
- 认证: 供应商portal使用独立token认证(简化版)

=== 技术债务/风险 ===
- 现有数据库需新增大量协同相关表
- 需要新增API路由: 约15个新路由文件
- 需要新增前端页面: 供应商portal、评审管理、寻源管理等

=== US-201: 寻源项目发布 ===
- 采购侧 SRM/寻源系统通过接口同步项目主数据（project_name, type, materials, deadline, invited_suppliers, evaluation_criteria）
- 本系统仅负责发布项目给受邀供应商，供应商在门户查看受邀项目并进入后续响应流程
- 已实现 API: POST /api/sourcing/projects 创建寻源项目（自动生成 SourcingInvitation 记录）
- 已实现 API: GET /api/sourcing/projects/invitations 供应商查看收到的寻源邀请
- 数据模型: SourcingProject, SourcingInvitation
- 经验：
  - 采购侧不在本系统内直接操作，仅同步数据
  - 供应商端可见受邀项目，后续响应流程由 SRM 系统主导
  - SourcingProject.status: DRAFT/OPEN/EVALUATING/AWARDED/CANCELLED
  - SourcingInvitation.status: pending/accepted/declined
  - 供应商主数据需提前同步，invited_supplier_ids 必须存在于合格供应商库
  - API 设计需兼容 SRM/ERP 系统批量同步
  - 仅保留菜单入口的 SRM 功能（如投标、开标）不做具体实现

- 寻源项目: SourcingProject/SourcingInvitation/SourcingBid/SourcingAward
- 合同管理: ContractTemplate/Contract/ContractComment
- 核心API: /api/sourcing/*, /api/contracts/*

=== Phase 2+ 角色边界调整 ===
- 从 Phase 2 开始，采购侧默认通过 SRM/ERP/WMS/QMS/财务系统同步数据到本系统，不在本系统中执行常规业务操作
- 供应商为门户侧主要操作方，负责响应邀请、投标、反馈合同、签署、确认计划、提交ASN、结算与发票等动作
- 采购侧在本系统中的例外操作: US-308 送货计划审批、US-309 装箱单与批次信息审批、US-402 结算单审核、US-404 发票审批
- 公告栏属于采购侧需要在本系统中维护的通用运营能力，不纳入以上默认规则

=== 通用能力: 公告栏运营协同 (US-501) ===
- 采购侧在本系统内执行: 发布、编辑、删除公告
- 供应商在本系统内执行: 查看公告、记录已读
- 外部系统同步来源: 无，公告栏数据完全由本系统维护
- 现有能力已覆盖: 公告列表、详情、发布、编辑、删除、已读记录、类型统计
