# 经验日志 - 供应商全链路协同平台

## Phase 0: 需求分析与架构规划

=== 项目启动 ===
- 项目名称: 白酒供应链协同平台 - 供应商全链路协同
- 分5大阶段: 供应商准入 / 寻源合同 / 订单执行 / 财务结算
- 当前Phase: Phase 1 (供应商准入)
- 已有资产: React前端(Ant Design)、FastAPI后端、SQLite数据库

=== 架构决策 ===
- 采用前后端分离架构
  - 前端: React + Ant Design + Vite (现有)
  - 后端: FastAPI + SQLAlchemy + SQLite (现有)
  - 协同API: /api/supplier-portal/*, /api/qualification/*, /api/sourcing/*, /api/forecast/*, /api/asn/*, /api/settlement/*, /api/invoices/*, /api/payments/*
- 数据库: 复用现有SQLite，新增协同相关表
- 认证: 供应商portal使用独立token认证(简化版)

=== 技术债务/风险 ===
- 现有数据库需新增大量协同相关表
- 需要新增API路由: 约15个新路由文件
- 需要新增前端页面: 供应商portal、评审管理、寻源管理等

## Phase 1: US-101~US-108 (供应商准入)

=== US-101: 供应商注册邀请 ===
- 核心表: supplier_invitations (invitation_code PK, supplier_name, email, status, expiry_date, created_by)
- API: POST /api/supplier-portal/invitations, GET /api/supplier-portal/invitations/{code}/validate
- 邀请码格式: 8位随机字符串
- 状态枚举: pending/accepted/expired

=== US-102: 供应商自助注册 ===
- 注册信息独立表: supplier_registrations (独立于主suppliers表)
- 审核通过后写入suppliers表

=== US-101 实施经验 ===
- 新增模型: SupplierInvitation, SupplierRegistration, SupplierCertification, SupplierAlert, User
- 新增API路由: /api/supplier-portal/invitations, /register, /pending-audit, /supplier-alerts
- async SQLAlchemy 注意点: join/relationship访问需用selectinload()预加载
- 邀请码格式: 8位UUID hex段
- 状态枚举: InvitationStatus(PENDING/ACCEPTED/EXPIRED/CANCELLED)
- 注册审核通过后: 自动创建supplier主数据记录
- 资质预警: 使用days_before_expiry字段触发预警消息
- UUID: python的uuid.uuid4().hex[:8].upper()
