## 1. 产品概述

### 1.1 产品定位

**产品名称**：白酒供应链数字化管控平台（Demo版）

**一句话定位**：面向白酒企业的采购方-供应商协同门户Demo，展示从供应商准入到结算协同的关键业务能力，以及采购到入库的核心链路可视化能力。

**核心场景**：聚焦"供应商准入 → 寻源合同 → 订单执行 → 财务结算"协同主链，并通过数据中台串联采购、仓储、物流等内部链路，实现采购侧跨系统同步与供应商在线响应。

### 1.2 业务目标

| 目标维度 | 具体目标 | 量化指标 |
|---------|---------|---------|
| 业务可视化 | 实现采购到入库全链路数据可视化 | 核心链路节点100%覆盖 |
| 异常感知 | 异常事件自动预警、快速定位 | 预警响应时间 < 5分钟 |
| 决策支撑 | 为管理层提供实时业务看板 | 看板刷新延迟 < 10秒 |
| 协同效率 | 各环节信息实时共享、减少沟通成本 | 信息传递效率提升50% |

### 1.3 目标用户

| 用户角色 | 职责描述 | 核心诉求 |
|---------|---------|---------|
| **采购主管** | 统筹原料采购计划、把控供应商 | 实时掌握采购进度、到货预期 |
| **供应商协同专员** | 在供应商门户响应邀请、报价、签署、ASN、结算与发票 | 及时获得采购信息、在线反馈与处理业务 |
| **仓库管理员** | 执行入库、出库、盘点操作 | 高效完成收验货、库存准确 |
| **质检员** | 原料/成品质量检验 | 快速完成检验、及时反馈 |
| **生产调度员** | 协调生产计划与原料供应 | 原料及时到位、生产顺畅 |
| **财务/管理层** | 成本核算、运营决策 | 实时了解供应链运营数据 |

### 1.4 白酒行业特点

白酒供应链具有鲜明的行业特征，本Demo在设计中充分体现：

| 行业特点 | 说明 | Demo体现 |
|---------|------|---------|
| **原料季节性** | 高粱、小麦等原料有固定收购季节 | 采购计划关联季节日历 |
| **窖池依赖性** | 基酒产量依赖窖池状态 | 生产协同模块体现窖池管理 |
| **年份价值** | 基酒有年份概念，越陈越香 | 库存管理支持年份标注 |
| **包装多样化** | 产品SKU多（多种规格、度数） | 成品库管理支持多规格 |
| **渠道层级多** | 经销商→二批→终端 | Demo聚焦前端采购入库 |
| **食品安全严** | 白酒食品安全管控严格 | 质检流程完整、可追溯 |

### 1.5 供应商协同范围与边界

当前版本的系统定位为"采购方与供应商之间的协同门户"，不替代企业内部 SRM、ERP、WMS、QMS、财务系统。执行层面的用户故事定义以 `assets/prd.json` 为准，本 PRD 在本节同步说明采购端和供应商端的责任边界。

| 范围 | 采购端在本系统中的角色 | 供应商在本系统中的角色 | 外部系统来源 |
|---|---|---|---|
| Phase 1 供应商准入 | 采购端与供应商端均存在实质操作，如邀请、审核、评审、审批 | 注册、提交资质、响应澄清、补正材料 | 以本系统内流程为主 |
| Phase 2-4 主协同链 | 采购端默认不在本系统中做常规业务操作，主要同步内部系统结果 | 供应商是主要操作方，负责响应、报价、反馈、签署、确认、ASN、结算、发票等动作 | SRM、ERP、WMS、QMS、财务系统 |
| 通用运营能力 | 采购端在本系统中维护公告栏等运营能力 | 供应商查看公告并记录已读 | 由本系统内维护 |

| 采购端例外操作 | 说明 |

| US-308 | 采购方在本系统批准、拒绝、修改送货计划 |

## 2. 领域模型（DDD设计）

### 2.1 领域划分

根据业务职能与职责边界，将白酒供应链领域划分为**5个核心领域**：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        白酒供应链全局领域                            │
├─────────────┬─────────────┬─────────────┬─────────────┬─────────────┤
│  采购域     │   仓储域    │   物流域    │ 生产协同域  │  数据中台域 │
│ Procurement │  Warehouse  │   Logitics │  Production │ DataCenter  │
│    Domain   │   Domain    │   Domain   │    Domain   │   Domain    │
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────┘
```

### 2.2 子域与限界上下文

#### 2.2.1 采购域（Procurement Domain）

**通用域**：采购基础能力，支撑其他核心域

| 限界上下文 | 职责 | 核心能力 |
|-----------|------|---------|
| 供应商管理上下文 | 供应商全生命周期管理 | 供应商准入、评级、合作状态管理 |
| 采购计划上下文 | 年度/月度采购计划制定 | 需求汇总、计划审批、计划调整 |
| 采购执行上下文 | 采购订单全流程管理 | 询价比价、下单、跟踪、入库协同 |

**核心实体**：供应商(Supplier)、采购计划(PurchasePlan)、采购订单(PurchaseOrder)

**值对象**：供应商评级(SupplierRating)、报价单(QuoteSheet)、交货期(DeliveryDate)

**聚合根**：

```
聚合根：供应商(Supplier)
├── 属性：supplier_id, name, type, contact_info, bank_account, status
├── 实体：SupplierRating (历史评级记录)
└── 领域服务：准入评估、信用评估

聚合根：采购订单(PurchaseOrder)
├── 属性：order_id, supplier_id, created_by, created_at, status
├── 实体：OrderLineItem (明细行)
├── 值对象：PaymentTerm, DeliveryTerm
└── 领域事件：OrderCreated, OrderConfirmed, OrderShipped, OrderArrived
```

---

#### 2.2.2 仓储域（Warehouse Domain）

**核心域**：仓储管理的核心能力

| 限界上下文 | 职责 | 核心能力 |
|-----------|------|---------|
| 库房管理上下文 | 库房、库区、库位管理 | 库房配置、库位规划、库容监控 |
| 库存管理上下文 | 库存的出入存管理 | 入库、出库、盘点、调拨、库存冻结 |
| 质检协同上下文 | 质量检验流程管理 | 送检、检验、判定、不合格处理 |

**核心实体**：库房(Warehouse)、库位(StorageLocation)、批次(Batch)、质检单(QualityInspection)

**值对象**：批次号(BatchNumber)、存储条件(StorageCondition)、有效期(ExpiryDate)

**聚合根**：

```
聚合根：批次(Batch)
├── 属性：batch_id, material_type, quantity, unit, production_date, expiry_date
├── 实体：QualityRecord (质检记录)
├── 值对象：StorageLocation (存储位置)
└── 领域事件：BatchCreated, BatchInspected, BatchStored

聚合根：质检单(QualityInspection)
├── 属性：inspection_id, batch_id, sample_size, inspector, status
├── 实体：InspectionResult (检验结果)
├── 值对象：QualityStandard, JudgmentResult
└── 领域事件：InspectionSubmitted, InspectionCompleted, InspectionPassed, InspectionFailed
```

---

#### 2.2.3 物流域（Logistics Domain）

**支撑域**：支持采购入库与成品出库

| 限界上下文 | 职责 | 核心能力 |
|-----------|------|---------|
| 运输管理上下文 | 在途运输监控 | 车辆调度、轨迹跟踪、到货预测 |
| 签收管理上下文 | 货物签收确认 | 到货通知、签收录入、差异处理 |

**聚合根**：

```
聚合根：运单(Waybill)
├── 属性：waybill_id, source_location, dest_location, carrier, vehicle
├── 实体：TrackingPoint (轨迹点)
├── 值对象：ETA, CargoInfo
└── 领域事件：ShipmentDeparted, ShipmentArrived, ShipmentSigned
```

---

#### 2.2.4 生产协同域（Production Domain）

**核心域**：生产与供应链的协同

| 限界上下文 | 职责 | 核心能力 |
|-----------|------|---------|
| 窖池管理上下文 | 窖池状态与产能管理 | 窖池档案、出酒率、酒龄管理 |
| 生产计划上下文 | 月度/周生产计划 | 计划排程、原料需求计算 |
| 领料协同上下文 | 生产领料与原料调拨 | 领料申请、库存锁定、实物调拨 |

**聚合根**：

```
聚合根：窖池(PitCellar)
├── 属性：pit_id, pit_name, capacity, status, start_date
├── 实体：FermentationRecord (发酵记录)
├── 值对象：BaseLiquorYield, AgingPeriod
└── 领域事件：PitActivated, PitDrained

聚合根：生产工单(ProductionOrder)
├── 属性：order_id, pit_id, planned_quantity, status
├── 实体：MaterialRequisition (领料单)
├── 值对象：ProductionSchedule
└── 领域事件：OrderPlanned, OrderStarted, OrderCompleted
```

---

#### 2.2.5 数据中台域（Data Platform Domain）

**通用域**：数据采集、治理与服务的共享平台

| 限界上下文 | 职责 | 核心能力 |
|-----------|------|---------|
| 数据采集上下文 | 业务系统数据接入 | CDC同步、批量导入、API接入 |
| 数据治理上下文 | 数据清洗、标准化、质量管理 | 主数据管理、数据清洗、血缘追踪 |
| 数据服务上下文 | 指标计算、可视化、报表 | 实时指标计算、看板配置、告警管理 |

**核心实体**：数据源(DataSource)、指标(Indicator)、看板(Dashboard)、告警规则(AlertRule)

**值对象**：数据质量评分(QualityScore)、刷新周期(RefreshCycle)

## 6. 非功能需求

### 6.1 性能需求

| 指标类型 | 要求 | 说明 |
|---------|------|------|
| **页面加载** | 首屏加载 < 3秒 | Dashboard首次加载时间 |
| **数据刷新** | 看板数据刷新 < 10秒 | 实时看板数据更新延迟 |
| **接口响应** | P95 < 2秒 | API接口响应时间 |
| **并发支持** | 同时在线用户 ≥ 50人 | Demo演示场景 |
| **数据同步** | CDC延迟 < 5秒 | 业务数据到看板延迟 |
| **系统可用性** | ≥ 99.5% | 月度可用时间 |

---

### 6.2 安全需求

| 安全维度 | 要求 |
|---------|------|
| **身份认证** | 用户名密码认证、登录日志记录 |
| **权限控制** | 基于角色的权限管理（RBAC） |
| **数据安全** | 敏感数据脱敏展示 |
| **传输安全** | HTTPS加密传输 |
| **审计日志** | 关键操作日志记录 |

---

### 6.3 可扩展性需求

| 扩展维度 | 设计要求 |
|---------|---------|
| **功能扩展** | 模块化架构，支持新功能快速接入 |
| **数据接入** | 标准化数据接口，支持新数据源快速对接 |
| **指标扩展** | 指标体系支持灵活配置和扩展 |
| **可视化扩展** | 看板支持自定义配置和新增 |

---

### 6.4 技术选型建议

| 技术层级 | 推荐技术 | 说明 |
|---------|---------|------|
| **前端框架** | Vue3 + Element Plus | 企业级UI组件库 |
| **可视化引擎** | ECharts / Apache ECharts | 图表渲染 |
| **实时通信** | WebSocket | 实时数据推送 |
| **消息队列** | Apache Kafka | CDC数据采集 |
| **实时计算** | Apache Flink | 实时指标计算 |
| **数据存储** | MySQL + Elasticsearch | 业务数据+搜索 |
| **缓存层** | Redis | 热数据缓存 |
| **运维监控** | Prometheus + Grafana | 系统监控 |
