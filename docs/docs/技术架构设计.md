## 1. 系统概述

### 1.1 项目定位

**白酒供应链数字化管控平台Demo版**，面向白酒企业客户展示数据中台可视化能力，聚焦"采购到入库"核心链路。

### 1.2 系统边界

| 范围 | 说明 |
|------|------|
| **涵盖领域** | 采购域、仓储域、物流域、生产协同域、数据中台域 |
| **Demo功能** | 供应商管理、采购订单、到货收货、质检入库、库存管理、数据驾驶舱 |
| **展示链路** | 采购计划 → 订单创建 → 供应商发货 → 物流运输 → 到货签收 → 质检入库 → 库存管理 |

### 1.3 系统规模

| 指标 | 预估规模 |
|------|---------|
| 用户规模 | Demo展示：≤50并发 |
| 日均订单 | Demo模拟：≤1000条/天 |
| 数据存储 | Demo数据：≤100万条记录 |
| 响应要求 | 看板刷新 < 10秒，API响应 < 500ms |

## 2. 技术栈选型

### 2.1 前端技术栈

| 技术 | 选型 | 说明 |
|------|------|------|
| **框架** | React 18 + TypeScript | 组件化开发，强类型支持 |
| **UI库** | Ant Design 5.x | 企业级React UI组件库 |
| **状态管理** | Zustand / React Query | 轻量状态管理 + 服务端状态 |
| **路由** | React Router 6 | 单页应用路由 |
| **图表** | ECharts / @ant-design/charts | 数据可视化 |
| **构建工具** | Vite | 快速开发体验 |
| **HTTP客户端** | Axios + axios-fetch-adapter | REST API调用 |

**选型理由**：
- React + TypeScript 提供开发效率和代码质量保障
- Ant Design 与 UI原型设计风格一致，减少样式开发成本
- ECharts 支持复杂数据可视化，适合驾驶舱大屏
- Vite 提供极快的冷启动和热更新

### 2.2 后端技术栈

| 技术 | 选型 | 说明 |
|------|------|------|
| **语言** | Python 3.11+ | 简洁高效，数据处理能力强 |
| **框架** | FastAPI | 高性能异步API框架 |
| **ORM** | SQLAlchemy 2.0 | 数据库ORM，支持异步 |
| **迁移** | Alembic | 数据库版本管理 |
| **验证** | Pydantic v2 | 数据验证和序列化 |
| **异步** | asyncio + uvicorn | ASGI异步服务器 |

**选型理由**：
- Python + FastAPI 开发效率高，适合Demo快速迭代
- FastAPI 自动生成OpenAPI文档，便于前后端对接
- SQLAlchemy 支持复杂查询和事务管理
- Pydantic 提供强大的数据验证能力

### 2.3 数据存储层

| 组件 | 选型 | 说明 |
|------|------|------|
| **主数据库** | PostgreSQL 15 | 关系型数据库，JSON支持 |
| **缓存** | Redis 7 | 数据缓存、会话存储 |
| **消息队列** | RabbitMQ (可选) | 领域事件异步处理 |

**选型理由**：
- PostgreSQL 支持复杂的业务查询和 JSON 类型
- Redis 提供高速缓存，减少数据库压力
- RabbitMQ 用于领域事件解耦（Demo可选）

## 4. 数据库设计

### 4.1 设计原则

1. **DDD聚合根映射**：每个聚合根对应1个核心表 + N个关联表
2. **值对象内联**：简单的值对象作为字段内联到主表
3. **实体独立表**：内部实体单独建表，通过外键关联
4. **事件追溯**：领域事件存储到事件表，支持业务审计

### 4.2 ER关系图（核心表）

```

└─────────────┘  1:N  │  (采购订单) │  1:N  └─────────────┘
```

### 4.3 数据表详细设计

#### 4.3.1 采购域表

**供应商表 (supplier)**

```sql
CREATE TABLE supplier (
    -- 主键
    supplier_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    supplier_code VARCHAR(50) NOT NULL UNIQUE,
    
    -- 基本信息
    name VARCHAR(200) NOT NULL,
    type VARCHAR(20) NOT NULL,  -- 综合供应商/粮食供应商/包材供应商
    category VARCHAR(100),       -- 主营品类
    business_license VARCHAR(50),
    
    -- 联系信息
    contact_person VARCHAR(100),
    contact_phone VARCHAR(50),
    contact_email VARCHAR(100),
    
    -- 银行信息
    bank_name VARCHAR(200),
    bank_account VARCHAR(100),
    
    -- 地址信息 (JSON存储)
    address JSONB DEFAULT '{}',
    
    -- 状态与评级
    status VARCHAR(20) DEFAULT '待审核',  -- 待审核/合作中/暂停/终止
    credit_level VARCHAR(10) DEFAULT 'C',  -- A/B/C
    
    -- 审计字段
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- 索引
    CONSTRAINT supplier_code_unique UNIQUE (supplier_code)
);

CREATE INDEX idx_supplier_status ON supplier(status);
CREATE INDEX idx_supplier_type ON supplier(type);
```

**供应商评级历史表 (supplier_rating_history)**

```sql
CREATE TABLE supplier_rating_history (
    rating_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    supplier_id UUID NOT NULL REFERENCES supplier(supplier_id),
    rating_year INTEGER NOT NULL,
    rating_level VARCHAR(10) NOT NULL,  -- A/B/C
    rating_date TIMESTAMP NOT NULL,
    rating_by VARCHAR(100),
    remarks TEXT,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_rating_supplier ON supplier_rating_history(supplier_id);
```

**供应商资质表 (supplier_qualification)**

```sql
CREATE TABLE supplier_qualification (
    qualification_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    supplier_id UUID NOT NULL REFERENCES supplier(supplier_id),
    type VARCHAR(50) NOT NULL,  -- 营业执照/生产许可证/质检报告
    file_url VARCHAR(500),
    expire_date DATE,
    status VARCHAR(20) DEFAULT '有效',  -- 有效/过期
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP

CREATE INDEX idx_qual_supplier ON supplier_qualification(supplier_id);
```

**采购订单表 (purchase_order)**

```sql
CREATE TABLE purchase_order (
    -- 主键
    order_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_number VARCHAR(50) NOT NULL UNIQUE,
    
    -- 关联信息
    supplier_id UUID NOT NULL REFERENCES supplier(supplier_id),
    
    -- 订单信息
    plan_type VARCHAR(20),        -- 年度计划/月度计划/临时计划
    season_tag VARCHAR(20),       -- 春季/夏季/秋季/冬季
    
    -- 金额信息
    total_amount DECIMAL(18, 2) DEFAULT 0,
    currency VARCHAR(10) DEFAULT 'CNY',
    
    -- 条款信息 (JSON存储)
    payment_term JSONB,
    delivery_term JSONB,
    
    -- 时间信息
    expected_delivery_date DATE,
    actual_delivery_date DATE,
    
    -- 状态与审批
    status VARCHAR(20) DEFAULT '草稿',  -- 草稿/已提交/已确认/已发货/已到货/已入库/已取消
    created_by VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    approved_by VARCHAR(100),
    approved_at TIMESTAMP,
    
    -- 备注
    remarks TEXT,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_order_number ON purchase_order(order_number);
CREATE INDEX idx_order_supplier ON purchase_order(supplier_id);
CREATE INDEX idx_order_status ON purchase_order(status);
CREATE INDEX idx_order_created ON purchase_order(created_at);
```

**订单明细表 (order_line_item)**


#### 4.3.2 仓储域表

**批次表 (batch)**

```sql
CREATE TABLE batch (
    -- 主键
    batch_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    batch_number VARCHAR(50) NOT NULL UNIQUE,
    
    -- 来源信息
    source_type VARCHAR(30),      -- 采购入库/生产入库/调拨入库/退料入库
    source_order_id UUID,        -- 来源订单ID
    
    -- 物料信息
    material_id UUID,
    material_name VARCHAR(200) NOT NULL,
    material_code VARCHAR(50),
    
    -- 批次属性 (白酒行业特色)
    production_date DATE,         -- 生产日期/收购日期
    harvest_season VARCHAR(20),    -- 春收/秋收
    origin_region VARCHAR(100),    -- 原料产地
    specification VARCHAR(200),    -- 规格等级
    starch_content DECIMAL(6, 2), -- 淀粉含量%
    moisture_content DECIMAL(6, 2), -- 水分含量%
    base_liquor_year INTEGER,     -- 基酒年份
    
    -- 数量与存储
    quantity DECIMAL(18, 4) NOT NULL,
    unit VARCHAR(20),
    storage_location_id UUID,
    storage_start_date TIMESTAMP,
    expiry_date DATE,
    
    -- 状态与质量
    current_status VARCHAR(20) DEFAULT '待检',  -- 待检/检验中/合格/不合格/已出库/已冻结
    quality_level VARCHAR(20),    -- 优等/一等/合格/不合格
    
    -- 追溯信息
    trace_code VARCHAR(100),       -- 追溯码
    remarks TEXT,
    
    -- 审计字段
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_batch_number ON batch(batch_number);
CREATE INDEX idx_batch_material ON batch(material_id);
CREATE INDEX idx_batch_status ON batch(current_status);
CREATE INDEX idx_batch_trace ON batch(trace_code);
CREATE INDEX idx_batch_expiry ON batch(expiry_date);
```

**质检表 (quality_inspection)**

```sql
CREATE TABLE quality_inspection (
    -- 主键
    inspection_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    inspection_number VARCHAR(50) NOT NULL UNIQUE,
    
    -- 关联批次
    batch_id UUID NOT NULL REFERENCES batch(batch_id),
    batch_number VARCHAR(50),
    
    -- 物料信息
    material_name VARCHAR(200),
    material_code VARCHAR(50),
    
    -- 质检类型与信息
    inspection_type VARCHAR(30),  -- 到货检验/定期检验/复检/出库检验
    sample_size INTEGER,
    sample_code VARCHAR(50),
    sampling_location VARCHAR(200),
    sampling_time TIMESTAMP,
    
    -- 检验人员
    inspector VARCHAR(100),
    inspector_assist VARCHAR(100),
    
    -- 判定结果
```

#### 6.2.1 订单履约服务 (OrderFulfillmentService)

```python
class OrderFulfillmentService:
    """订单履约服务 - 协调订单全生命周期"""
    
    async def track_order_fulfillment(self, order_id: UUID) -> FulfillmentStatus:
        """跟踪订单履约状态"""
        order = await self.order_repo.get_by_id(order_id)
        
        # 计算履约进度
        progress = self.calculate_progress(order)
        
        # 检查异常
        anomalies = self.check_anomalies(order)
        
        # 预测交付
        eta = self.predict_delivery(order)
        
        return FulfillmentStatus(
            order_id=order_id,
            progress=progress,
            anomalies=anomalies,
            estimated_arrival=eta
        )
    
    async def handle_shipment_event(self, event: OrderShippedEvent):
        """处理发货事件"""
        order = await self.order_repo.get_by_id(event.order_id)
        order.update_shipment(event.shipped_items)
        
        # 创建运单
        waybill = await self.waybill_service.create_waybill(order)
        
        # 发送通知
        await self.notification_service.send_shipment_notification(order)
        
        await self.order_repo.save(order)
```

#### 6.2.2 批次追溯服务 (BatchTraceService)

```python
class BatchTraceService:
    """批次追溯服务 - 实现全链路追溯"""
    
    async def trace_batch(self, batch_id: UUID) -> TraceChain:
        """追溯批次全生命周期"""
        batch = await self.batch_repo.get_by_id(batch_id)
        
        # 获取来源追溯
        source_trace = await self.get_source_trace(batch)
        
        # 获取质检追溯
        quality_trace = await self.get_quality_trace(batch)
        
        # 获取库存事务追溯
        inventory_trace = await self.get_inventory_trace(batch)
        
        # 获取出库追溯
        outbound_trace = await self.get_outbound_trace(batch)
        
        return TraceChain(
            batch=batch,
            source=source_trace,
            quality=quality_trace,
            inventory=inventory_trace,
            outbound=outbound_trace
        )
    
    async def generate_trace_code(self, batch: Batch) -> str:
        """生成追溯码"""
        # 格式：B-年份-批次号-校验码
        return f"B-{batch.production_date.year}-{batch.batch_number[-4:]}-{self.generate_check_digit()}"
```

#### 6.2.3 质检判定服务 (InspectionJudgmentService)

```python
class InspectionJudgmentService:
    """质检判定服务 - 智能质检判定"""
    
    async def auto_judge(self, inspection_id: UUID) -> JudgmentResult:
        """自动判定质检结果"""
        inspection = await self.inspection_repo.get_by_id(inspection_id)
        items = await self.inspection_item_repo.get_by_inspection(inspection_id)
        
        # 按类别分组检验项目
        grouped_items = self.group_by_category(items)
        
        # 逐类判定
        judgments = []
        for category, category_items in grouped_items.items():
            judgment = self.judge_category(category_items)
            judgments.append(judgment)
        
        # 综合判定
        final_result = self.synthesize_judgments(judgments)
        
        return final_result
    
    def judge_category(self, items: List[InspectionItem]) -> CategoryJudgment:
        """按类别判定"""
        critical_failed = any(i.is_critical and not i.is_passed for i in items)
        normal_failed = any(not i.is_critical and not i.is_passed for i in items)
        
        if critical_failed:
            return CategoryJudgment(category=items[0].category, result="不合格")
        elif normal_failed:
            return CategoryJudgment(category=items[0].category, result="让步接收")
        return CategoryJudgment(category=items[0].category, result="合格")
```

#### 6.2.4 指标计算服务 (MetricsCalculationService)

```python
class MetricsCalculationService:
    """指标计算服务 - 数据中台核心"""
    
    # 预定义指标
    METRICS = {
        "purchase_order_count": PurchaseOrderCount,
        "purchase_amount": PurchaseAmount,
        "inbound_quantity": InboundQuantity,
        "inspection_pass_rate": InspectionPassRate,
        "inventory_turnover": InventoryTurnover,
        "supplier_delivery_rate": SupplierDeliveryRate,
    }
    
    async def calculate_indicator(
        self, 
        indicator_code: str, 
        date_range: DateRange
    ) -> IndicatorValue:
        """计算单个指标"""
        metric_class = self.METRICS.get(indicator_code)
        if not metric_class:
            raise ValueError(f"Unknown indicator: {indicator_code}")
        
        metric = metric_class()
        value = await metric.calculate(date_range)
        
        return IndicatorValue(
            code=indicator_code,
            value=value,
            calculated_at=datetime.now()
        )
    
    async def calculate_dashboard_indicators(
        self, 
        dashboard_id: UUID
    ) -> Dict[str, IndicatorValue]:
        """计算看板全部指标（带缓存）"""
        # 尝试从Redis获取缓存
        cached = await self.redis.get(f"dashboard:{dashboard_id}")
        if cached:
            return cached
        
        dashboard = await self.dashboard_repo.get_by_id(dashboard_id)
        
        results = {}
        for indicator_config in dashboard.indicators:
            result = await self.calculate_indicator(
                indicator_config.code, 
                indicator_config.date_range
            )
            results[indicator_config.code] = result
        
        # 缓存30秒
        await self.redis.setex(
            f"dashboard:{dashboard_id}", 
            30, 
            results
        )
        
        return results
```

```python
class AuditMiddleware:
    """操作审计中间件"""
    
    async def __call__(self, request: Request, call_next):
        # 记录请求
        audit_log = AuditLog(
            user_id=self.get_user_id(request),
            action=request.method,
            path=request.url.path,
            method=f"{request.method} {request.url.path}",
            ip_address=request.client.host,
            user_agent=request.headers.get("user-agent"),
            request_body=await self.get_body(request),
            timestamp=datetime.now()
        )
        
        response = await call_next(request)
        
        # 记录响应
        audit_log.response_status = response.status_code
        audit_log.response_time = (datetime.now() - audit_log.timestamp).total_seconds()
        
        await self.audit_repo.save(audit_log)
        
        return response
```
