## 三、技术方案

### 3.1 拖拽库选型
**推荐: @dnd-kit**
- 理由：轻量、可访问性好、支持多平台
- 替代方案：react-beautiful-dnd (已停止维护)

### 3.2 组件结构
```
src/components/crm/
├── kanban-board.tsx      # 看板主组件
├── kanban-column.tsx     # 列组件
├── kanban-card.tsx       # 卡片组件
└── kanban-board-context.tsx  # 拖拽上下文

src/app/
├── opportunities/kanban/page.tsx  # 商机看板
└── leads/kanban/page.tsx          # 任务看板
```

### 3.3 数据流
```
用户拖拽 → onDragEnd → 更新本地状态 → 调用 CRM Context 更新方法 → 持久化
```
