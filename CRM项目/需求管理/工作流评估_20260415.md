# 工作流自动化功能评估报告

## 项目现状

### CRM 项目结构
- 技术栈：Next.js 16 + TypeScript + TailwindCSS + Shadcn/UI + Supabase
- 现有模块：客户、联系人、销售线索、商机、报价单、合同、回款计划、活动记录、跟进记录、通知
- 数据库：使用 Drizzle ORM 定义 schema，Supabase 作为后端

### 当前缺失
- **工作流自动化模块** - 不存在
- 触发器系统
- 自动动作执行
- 工作流模板管理
- 执行日志记录

---

## 功能需求分析

### 1. 触发条件
| 触发类型 | 描述 | 适用场景 |
|---------|------|---------|
| customer.created | 新建客户 | 自动发送欢迎邮件 |
| customer.updated | 客户信息变更 | 更新客户标签 |
| opportunity.stage_changed | 商机阶段变更 | 发送阶段通知 |
| opportunity.created | 新建商机 | 自动分配销售 |
| opportunity.updated | 商机变更 | 记录变更日志 |
| contract.signed | 合同签署 | 创建履约节点 |
| contract.created | 新建合同 | 通知财务 |
| payment.overdue | 回款逾期 | 触发催款流程 |
| followup.overdue | 跟进逾期 | 提醒待跟进 |

### 2. 自动动作
| 动作类型 | 描述 | 参数 |
|---------|------|------|
| send_email | 发送邮件 | 收件人、模板、主题、内容 |
| create_task | 创建任务 | 任务标题、负责人、截止日期 |
| update_field | 更新字段 | 实体类型、实体ID、字段名、新值 |
| add_tag | 添加标签 | 实体ID、标签名 |
| remove_tag | 移除标签 | 实体ID、标签名 |
| create_followup | 创建跟进 | 跟进类型、内容、计划时间 |
| send_notification | 发送通知 | 通知类型、标题、内容 |
| webhook | 触发 Webhook | URL、请求头、请求体 |

### 3. 工作流模板
| 模板名称 | 触发条件 | 执行动作 |
|---------|---------|---------|
| 新客户欢迎 | customer.created | 发送欢迎邮件 + 创建跟进任务 |
| 商机阶段通知 | opportunity.stage_changed | 发送通知给负责人 |
| 合同签署流程 | contract.signed | 创建履约节点 + 通知财务 |
| 逾期提醒 | payment.overdue | 发送催款通知 + 创建跟进 |

## 技术实现方案

### 数据库 Schema
workflows - 工作流定义表
workflow_triggers - 触发器配置表
workflow_actions - 动作配置表
workflow_executions - 执行记录表
workflow_execution_logs - 执行日志表

### API 设计
GET    /api/workflows           - 获取工作流列表
POST   /api/workflows           - 创建工作流
GET    /api/workflows/:id       - 获取工作流详情
PUT    /api/workflows/:id       - 更新工作流
DELETE /api/workflows/:id       - 删除工作流
POST   /api/workflows/:id/enable   - 启用工作流
POST   /api/workflows/:id/disable  - 禁用工作流
GET    /api/workflows/:id/executions - 获取执行记录
POST   /api/workflows/trigger   - 手动触发工作流
GET    /api/workflows/templates - 获取预设模板

### 前端页面
/workflows              - 工作流列表页
/workflows/new          - 创建工作流
/workflows/:id/edit     - 编辑工作流
/workflows/:id/logs      - 执行日志
